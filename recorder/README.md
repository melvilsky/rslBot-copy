# Запись кликов (позиция + цвет)

Документация описывает, как в проекте записываются клики мыши вместе с координатами и цветом пикселя. Используется для подбора координат под `pixel_check` / `await_click` и для отладки UI.

## Где реализовано

| Что | Файл |
|-----|------|
| Логика записи | `classes/Recorder.py` |
| Команды Telegram | `classes/App.py` (`record_on`, `record_off`) |

Ранний вариант `recorder/recorder.py` (только `pos`, без цвета) удалён. Остаётся один способ записи — через класс `Recorder`.

## Как пользоваться

1. Запустите бота и откройте окно игры в нужном состоянии (экран, на котором нужны точки).
2. В Telegram отправьте **`/record_on`** — начнётся запись.
3. Кликайте **левой кнопкой** по нужным элементам интерфейса.
4. Отправьте **`/record_off`** — запись остановится, в чат придёт текстовый отчёт, на диск сохранятся JSON и кропы.

Если `pynput` не установлен, бот ответит: `Recorder unavailable (pynput not installed)`.

## Что происходит при каждом клике

Обработчик `Recorder._on_click`:

1. **Слушатель** `pynput.mouse.Listener` ловит нажатие левой кнопки (`pressed=True`, `Button.left`).
2. Координаты округляются до целых: `x`, `y`.
3. Делается **полный скриншот экрана** (`pyautogui.screenshot()`), не отдельный `pixel()` — цвет берётся из того же кадра, что и координаты.
4. Считывается **RGB точки клика** — `rgb` в формате `[R, G, B]`.
5. Считается **средний RGB по области 5×5** вокруг точки (`avg_rgb_5x5`, радиус 2 пикселя) — устойчивее к антиалиасингу и шуму.
6. Сохраняется **кроп 120×120** вокруг клика (PNG) для визуальной проверки.
7. Событие добавляется в список `events` и пишется в лог.

При остановке (`stop`) все события сериализуются в JSON.

## Куда сохраняется результат

При `start()` создаётся сессия с именем `recording_YYYYMMDD_HHMMSS`:

```
recorder/
  recordings/
    recording_20260519_203045.json          # все клики
    recording_20260519_203045_assets/         # кропы по каждому клику
      click_0001_crop.png
      click_0002_crop.png
      ...
```

## Формат события в JSON

```json
{
  "x": 878,
  "y": 160,
  "rgb": [218, 0, 0],
  "avg_rgb_5x5": [215, 2, 1],
  "crop": "recorder/recordings/recording_20260519_203045_assets/click_0001_crop.png",
  "crop_region": [818, 100, 120, 120],
  "color": [218, 0, 0],
  "time": 1.23,
  "button": "left"
}
```

| Поле | Описание |
|------|----------|
| `x`, `y` | Координаты клика на экране |
| `rgb` | Цвет пикселя в точке клика `[R, G, B]` |
| `avg_rgb_5x5` | Средний цвет по окну 5×5 вокруг точки |
| `color` | То же, что `rgb` (кортеж в JSON станет массивом); оставлено для совместимости |
| `crop` | Путь к PNG-фрагменту вокруг клика |
| `crop_region` | `[left, top, width, height]` области кропа на скриншоте |
| `time` | Секунды от начала записи |
| `button` | Всегда `"left"` (записываются только ЛКМ) |

## Как перенести значения в координаты локации

Записанные `x`, `y` и `rgb` (или `avg_rgb_5x5`, если точка «шумная») можно перенести в JSON координат, см. [coordinates/README.md](../coordinates/README.md):

```json
{
  "claim_free_refill_coins": {
    "x": 878,
    "y": 160,
    "rgb": [218, 0, 0],
    "mistake": 20,
    "description": "Кнопка из записи кликов"
  }
}
```

Рекомендации:

- для проверки пикселя в боте обычно достаточно **`rgb`** из точки клика;
- если цвет «прыгает» на границах кнопки — возьмите **`avg_rgb_5x5`** и подберите `mistake` (часто 10–20);
- убедитесь, что игра в том же разрешении/масштабе, что при записи — координаты привязаны к экрану, а не к окну ОС отдельно.

## Зависимости

- **pynput** — глобальный слушатель мыши (`requirements.txt`)
- **pyautogui** — скриншот экрана
- **Pillow** — чтение пикселей и сохранение кропов

## Ограничения

- Записываются только **нажатия** левой кнопки (не отпускания, не ПКМ/СКМ).
- Координаты — **экранные** (как отдаёт `pynput`), в том же пространстве, что использует `pyautogui` при кликах бота.
- Цвет берётся с **текущего** кадра экрана в момент клика: если поверх точки анимация или курсор, RGB может отличаться от «чистого» UI.
- Клик вне границ скриншота приведёт к ошибке записи этого события (остальные клики сохранятся).
